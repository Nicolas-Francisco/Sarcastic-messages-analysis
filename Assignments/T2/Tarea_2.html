Para este clasificador utilizaremos Cross-validation. Con cross-validation, el conjunto de entrenamiento se divide en $k$ conjuntos disjuntos; se entrena sobre los datos correspondientes a $k-1$ de stos, y se evala sobre el conjunto restante. Esto se repite $k$ veces, evaluando siempre sobre un conjunto distinto. Teniendo en cuenta el parmetro $k$, a ste mtodo se le llama $k$-fold cross-validation.

El cdigo que est a continuacin realizar 10-fold cross-validation usando Decision Tree sobre los datos. La forma de estimar el rendimiento del clasificador es, entre otras cosas, calculando el promedio de todos los k-folds.

PREGUNTA 3.1¶

Para realizar la evaluacin de distintos clasificadores, vamos a crear la funcin run_classifier(), la cual evala un clasificador clf recibido como parmetro, un dataset X,y (features y target) y un nmero de tests llamado num_test. Esta funcin almacena y retorna los valores de precision, recall y f1-score en la variable metrics adems de los resultados de prediccin.

En base a lo anterior, incluya las sentencias que ajusten el modelo junto a su correspondiente prediccin sobre los datos. No use cross-validation ni tampoco el parmetro random_state.

In [9]:
### COMPLETAR ESTE CDIGO

## run_classifier recibe un clasificador y un dataset (X, y)
## y opcionalmente la cantidad de resultados que se quiere obtener del clasificador

from sklearn.metrics import f1_score, recall_score, precision_score
from sklearn.model_selection import train_test_split
import numpy as np

def run_classifier(clf, X, y, num_tests=100):
    metrics = {'f1-score': [], 'precision': [], 'recall': []}
    
    for _ in range(num_tests):
        X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=.30)

        
        
        clf.fit(X_train, y_train)    # se entrena
        predictions = clf.predict(X_test)  # se testea


        
        metrics['y_pred'] = predictions
        metrics['f1-score'].append(f1_score(y_test, predictions)) 
        metrics['recall'].append(recall_score(y_test, predictions))
        metrics['precision'].append(precision_score(y_test, predictions))
    
    return metrics

Parte 4: Seleccionar hiperparámetros¶

Los hiperparmetros son parmetros que no se aprenden directamente dentro de los estimadores. En scikit-learn se pasan como argumentos al constructor de las clases, por ejemplo cul kernel usar para Support Vector Classifier, o qu criterio para Decision Tree, etc. Es posible y recomendable buscar en el espacio de hiperparmetros la mejor alternativa.

Tenga en cuenta que es comn que un pequeo subconjunto de esos parmetros pueda tener un gran impacto en el rendimiento predictivo o de clculo del modelo, mientras que otros pueden dejar sus valores predeterminados. Se recomienda leer la documentacin de la clase de estimador para obtener una mejor comprensin de su comportamiento esperado.

GridSearchCV¶

Una alternativa para seleccionar hiperparmetros es GridSearchCV, la cual considera exhaustivamente todas las combinaciones de parmetros. GridSearchCV recibe un estimador, recibe param_grid (un diccionario o una lista de diccionarios con los nombres de los parametros a probar como keys y una lista de los valores a probar), scoring una o varias funciones de puntuacin (score) para evaluar cada combinacin de parametros (opciones vlidas: https://scikit-learn.org/stable/modules/model_evaluation.html) y cv una extrategia para hacer validacin cruzada.

El siguiente cdigo muestra cmo seleccionar el nmero de vecinos y qu pesos otorgar a los vecinos en un clasificador KNN.

Pregunta 5. 1¶

Por qu aplicar (sub/over) sampling de las clases sobre el conjunto de entrenamiento en lugar de aplicarlo sobre el dataset completo?

Si se aplicara sobre el data set completo se perdera la distribucin real del dataset respecto a la clase, pero si lo hacemos sobre el conjunto de entrenamiento, estamos entregandole al clasificador la misma cantidad de ambas clases, entrenando al clasificador de forma imparcial (esto es, que no elige ms de una clase sobre el otro debido a la cantidad entregada en el entrenamiento)

Nota: Libreras como imblearn son muy tiles para balancear los datos.

  • Indique una desventaja de usar oversampling y una desventaja de usar subsampling en clasificacin.

    Una desventaja de usar oversampling es que dejamos muchos menos casos de testing para la clase con menor cantidad en el dataset, llevando a que las mtricas sean menos fiables. Por otro lado, una desventaja de usar subsampling es que dejamos muchos datos de testing comparados con los de training en el caso de la clase con ms cantidad en el dataset, llevando a que el clasificador no est tan bien entrenado.
